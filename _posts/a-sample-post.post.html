2011-05-08 10:00 The mog post file format In this post I will tell you how the posts files should be written in mog. You will see how simple it is!

The file format mog uses is really, really simple. You only have to use markup wherever you want to, which usually will be in the post content.

The only thing you need to know is that the file will be splitted into four parts:

The publication time

The first line with text in the file will be parsed as the publication time of the post. It will handle any format that DateTime#parse would. Remember that if no timezone is specfied all times will be in GMT+00:00.

The title

The first line with text in the file will be taken as the post title. For example this post is entitled The mog post file format. The post can't be splitted in more than one line for obvious reasons.

You shouldn't use markup in the title. It should be plain old text.

The description

The second line with text will be taken as the post description. In the particular case of this post file it will be In this post I will tell you how the posts files should be written in mog. You will see how simple it is!

You shouldn't use markup in the description. Write it in plain text.

The content

The rest of the file will be considered the post content. You can use whatever markup language you want in it. Every markup language supported by Sinatra is supported by mog. You can even use plain HTML.

The only thing you have to take care of is ensuring that the file extension matches the markup you are using. For example: if you are using HAML the file extension should be file-name.post.haml
